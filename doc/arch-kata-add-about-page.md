# アーキテクチャ・カタ研修アプリ 追加機能：Aboutページ

## 追加機能の概要

アプリの目的・対象ユーザー・使い方を説明する `/about` ページを新規作成する。
トップページからリンクを追加する。

---

## ページ構成

以下のセクションを上から順に配置する。

### 1. ヒーローセクション
- バッジ：「Architecture Kata」
- 見出し：「ソフトウェアアーキテクチャを、実践で学ぶ」
- 説明文：アーキテクチャ・カタの概要を2〜3文で説明

### 2. 背景セクション
- 見出し：「なぜアーキテクチャの練習が必要か」
- Ted Newardの引用をハイライトボックスで表示
  - 「優れたアーキテクトを育てたいなら、アーキテクトにアーキテクチャを設計させるしかない。しかし設計の機会がキャリアに6回も訪れないとしたら、どうやって腕を磨けばいいのか？― Ted Neward」
- 引用の前後に説明文を配置

### 3. AI時代の文脈セクション
- 見出し：「AIが進化するほど、アーキテクチャ力が問われる」
- 「AIが担う」「人間が担う」を色分けしたバッジで対比表示
  - AIが担う（赤系バッジ）：コードの実装・テストの生成・ドキュメント作成など「正解が決まっている」作業
  - 人間が担う（緑系バッジ）：アーキテクチャの選択・トレードオフの判断・ビジネス要件との調整など「場合による」判断
- 補足説明文：アーキテクチャの選択に正解はなく、トレードオフを文脈に応じて判断する能力はAIには代替できないことを説明

### 4. 演習の流れセクション
- 見出し：「アーキテクチャ・カタで何をするか」
- 以下の5ステップをタイムライン形式（番号＋縦線＋内容）で表示

| ステップ | タイトル | 説明 |
|---------|---------|------|
| 1 | お題を読む | 架空のプロジェクト要件（RFP形式）を読み、システムの目的・制約を把握します。 |
| 2 | 要件を確認する | 設計を始める前に「何を確認すべきか」を考え、顧客への質問と想定回答を整理します。 |
| 3 | アーキテクチャ特性を選ぶ | 可用性・スケーラビリティ・セキュリティなど12の特性から最大3つを選び、優先順位と理由を言語化します。 |
| 4 | システム構成を考える | 選んだ特性を実現するために必要なコンポーネントを選び、構成の方針を説明します。 |
| 5 | 発表・議論する | 他のグループと回答を比較し、なぜ異なる判断になったかを議論します。正解はなく、説明できるかどうかが重要です。 |

### 5. 対象ユーザーセクション
- 見出し：「こんな方に使ってほしい」
- 以下の4ペルソナをグリッドカードで表示

| ペルソナ | 説明 |
|---------|------|
| 新卒・若手エンジニア | 実務でアーキテクチャを考える機会がまだ少ない方。設計判断の思考プロセスを体験できます。 |
| 中堅エンジニア | 実装は得意だが設計判断の言語化が苦手な方。トレードオフを整理する訓練になります。 |
| 研修担当・チームリーダー | チームのアーキテクチャ思考を底上げしたい方。グループワークの進行ツールとして使えます。 |
| アーキテクト志望 | 将来アーキテクトを目指すエンジニア。繰り返し練習することで設計判断の引き出しが増えます。 |

### 6. 学べることセクション
- 見出し：「このアプリで身につく力」
- 以下の4項目をグリッドカードで表示

| タイトル | 説明 |
|---------|------|
| トレードオフ思考 | 何かを得れば何かを失う。その判断を根拠とともに説明できるようになります。 |
| 要件定義の入口 | 設計の前に何を確認すべきかを考える習慣が身につきます。 |
| 技術的な議論力 | チームで設計について話し合い、異なる判断の理由を理解できるようになります。 |
| 設計の言語化 | なぜその構成を選んだかをステークホルダーに説明できる力が鍛えられます。 |

### 7. CTAセクション
- 説明文：「まずはお題を選んで、グループワークを始めてみましょう。」
- ボタン：「研修を始める」→ トップページ（`/`）へ遷移

## デザイン仕様

### 全体
- 最大幅：720px・中央揃え
- フォント：Tailwindのデフォルト（sans-serif）
- カラー：Tailwindのユーティリティクラスを使用
- セパレーター：セクション間に `<hr>` を配置

### ヒーローセクション
- バッジ：`bg-blue-50 text-blue-600` の丸みのあるピル型
- 見出し：`text-3xl font-medium` 中央揃え
- テキスト全体：中央揃え・上下パディング大

### 引用ハイライトボックス
- `border-l-2 border-blue-400 bg-gray-50 px-5 py-4 rounded-r-lg`
- テキスト：`text-gray-700 leading-relaxed`

### AI対比ボックス
- 背景：`bg-gray-50 rounded-xl p-5`
- AIバッジ：`bg-red-50 text-red-600 text-xs font-medium px-3 py-1 rounded-full`
- 人間バッジ：`bg-green-50 text-green-600 text-xs font-medium px-3 py-1 rounded-full`

### タイムライン
- 番号：円形（`rounded-full`）・`bg-gray-100 text-gray-500`
- 縦線：`w-px bg-gray-200` で番号と番号を繋ぐ
- 最後のステップは縦線なし

### グリッドカード
- `grid grid-cols-1 sm:grid-cols-2 gap-3`
- 各カード：`bg-gray-50 rounded-xl p-4`
- タイトル：`text-sm font-medium`
- 説明：`text-sm text-gray-500 leading-relaxed`

### CTAセクション
- 上部にボーダー・中央揃え
- ボタン：`border border-gray-200 rounded-lg px-6 py-2.5 text-sm font-medium hover:bg-gray-50`

## トップページへの追加

`/`（トップページ）のフッター部分に以下のリンクを追加してください。

```
このアプリについて →　/about へのリンク
```

---

## ファイル構成

```
app/
  about/
    page.tsx    ← 新規作成
```

---

## 実装手順

### Step 1：Aboutページ作成
```
app/about/page.tsx を新規作成してください。
上記のセクション構成・デザイン仕様に従いTailwind CSSで実装してください。
Next.js App RouterのServer Componentとして作成します（'use client'不要）。
```

### Step 2：トップページにリンク追加
```
app/page.tsx のフッター部分に「このアプリについて」リンクを追加してください。
Next.jsの<Link>コンポーネントで /about へ遷移させてください。
```
